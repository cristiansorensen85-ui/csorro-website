# CSorro OS Product Principles

CSorro OS exists to reduce the mental load of running creative work.

## 1. Reduce mental load
Every screen should answer: what matters, what is blocked, and what should I do next?

## 2. One purpose per page
Mission Control = attention. Workspaces = company context. Projects = work delivery. People = access and responsibility. Assets = storage and approval. CORE = fewer clicks.

## 3. No duplicate navigation
Do not show workspace switching in several places. Keep workspace switching in the sidebar/dropdown and use page space for useful context.

## 4. Progressive disclosure
Show the essentials first. Put advanced actions behind drawers, modals, tabs or More menus.

## 5. Private by default
Workspaces, projects, assets, whiteboards, chats and profiles are private unless explicitly shared or published.

## 6. Everything connects
Tasks, chat, files, whiteboards, approvals and activity should link to each other. Nothing should disappear after saving.

## 7. CORE saves clicks
CORE must help users act, not just read. If CORE adds noise, simplify it.

## 8. Consistency over novelty
New pages must use the same sidebar, header, card, button, modal, form and spacing system.
